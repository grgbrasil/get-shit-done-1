# Milestone v1.0: GSD Impact Analysis

**Status:** SHIPPED 2026-04-01
**Phases:** 1-7
**Total Plans:** 16

## Overview

Sistema de memória global e análise de impacto para o GSD. Resolve o problema de amnésia entre planos/fases e de efeitos colaterais silenciosos quando executores modificam funções compartilhadas.

## Phases

### Phase 1: function-map (2 plans)

**Goal:** Flat JSON function map com O(1) key lookup para qualquer função do projeto.

Plans:
- [x] 01-01: fmap.cjs CRUD module + gsd-tools dispatcher + model-profiles
- [x] 01-02: gsd-cataloger agent + changed-files incremental scanning

### Phase 2: impact-analysis (2 plans)

**Goal:** Análise de impacto pré-edição usando o function map.

Plans:
- [x] 02-01: fmap impact subcommand (caller/signature snapshots + normalizeSignature)
- [x] 02-02: PreToolUse advisory hook + executor protocol para mid-execution impact analysis

### Phase 3: model-routing-integration (3 plans)

**Goal:** Integrar function map e impact analysis no pipeline de execução.

Plans:
- [x] 03-01: model_overrides e impact_analysis config defaults com three-level merge
- [x] 03-02: ContextFiles functionMapStats field + auto-generate para planner/executor
- [x] 03-03: Post-wave cataloger em execute-phase + impact analysis opt-in

### Phase 4: pre-flight-dependency-resolver (2 plans)

**Goal:** Resolver dependências de fase automaticamente antes de execução.

Plans:
- [x] 04-01: Preflight CLI (CONTEXT.md, UI-SPEC.md, dependent phases, plans existence)
- [x] 04-02: Preflight gates em plan-phase, execute-phase, ui-phase workflows

### Phase 5: ops-foundation-registry (2 plans)

**Goal:** Registry de áreas operacionais com auto-detecção.

Plans:
- [x] 05-01: OPS registry CRUD com hybrid area auto-detection (routes + directories)
- [x] 05-02: Fix followImports para manual/directory areas

### Phase 6: ops-workflows (3 plans)

**Goal:** Workflows operacionais com contexto de planejamento.

Plans:
- [x] 06-01: appendHistory, computeBlastRadius, refreshTree, cmdOpsSummary helpers
- [x] 06-02: cmdOpsInvestigate + cmdOpsDebug context-pack
- [x] 06-03: Blast-radius dispatch para ops:feature e ops:modify

### Phase 7: ops-governance (2 plans)

**Goal:** Status, specs e backlog per-area.

Plans:
- [x] 07-01: cmdOpsStatus (health scoring) + cmdOpsSpec (show/edit/add)
- [x] 07-02: cmdOpsBacklog (list/add/prioritize/promote/done)

## Milestone Summary

**Key Decisions:**
- Flat JSON over SQLite for function map (git-diffable, no deps)
- Serena MCP as primary provider, LLM-assisted grep as fallback
- Advisory hook (non-blocking) for impact analysis
- Hybrid area detection (routes + directory scanning) for ops registry

- v1.0 audit found MODEL-01 through MODEL-04 unsatisfied (carried to v2.0)
- INT-01 integration gap noted
