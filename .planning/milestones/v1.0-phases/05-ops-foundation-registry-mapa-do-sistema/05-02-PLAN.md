Add internal helpers:

`writeTreeJson(cwd, slug, tree)`:
- Calls `ensureAreaDir(cwd, slug)` to create `.planning/ops/{slug}/` if missing
- Writes `path.join(areaDir(cwd, slug), 'tree.json')` with `JSON.stringify(tree, null, 2)`

`readTreeJson(cwd, slug)`:
- Returns parsed tree.json or null if not exists

`buildNodeId(type, name)`:
- Returns `type + ':' + name` (e.g., `"route:/users"`, `"view:UsersView"`, `"component:UserTable"`)

`scanImports(filePath, projectFiles)`:
- Reads the file content via `fs.readFileSync`
- Extracts import/require statements using regex:
  - ES6: `/import\s+.*\s+from\s+['"]([^'"]+)['"]/g`
  - CommonJS: `/require\(\s*['"]([^'"]+)['"]\s*\)/g`
  - PHP use: `/use\s+([A-Z][A-Za-z\\]+)/g`
- Resolves relative paths against the file's directory
- Returns array of resolved file paths that exist in projectFiles

`classifyFileType(filePath)`:
- Returns node type based on file path heuristics:
  - Path contains `/route` or `/router` -> `"route"`
  - Path contains `/view` or `/views` or `/pages` -> `"view"`
  - Path contains `/component` or `/components` -> `"component"`
  - Path contains `/controller` or `/endpoint` or `/api` -> `"endpoint"`
  - Path contains `/service` or `/services` -> `"service"`
  - Path contains `/model` or `/models` or `/Model` -> `"model"`
  - Path contains `/migration` or has `.sql` extension -> `"table"`
  - Default: `"component"`

`extractNodeName(filePath)`:
- Returns the filename without extension (e.g., `UserController.php` -> `UserController`)

**cmdOpsMap(cwd, area, raw) implementation:**
1. If !area, `error('Usage: gsd-tools ops map <area>')`
2. Slugify area input
3. Read registry, find area entry by slug. If not found, `error('Area not found in registry: ' + slug)`
4. Get project files via `listProjectFiles(cwd)`
5. Find files belonging to this area:
   - For route-detected areas: start from the route file, follow imports recursively (max 3 levels deep)
   - For directory-detected areas: find files whose path contains the area slug or area name
   - For manual areas: find files whose path contains the area slug
6. Build nodes array: For each found file, create node per D-11: `{ id: buildNodeId(classifyFileType(f), extractNodeName(f)), type: classifyFileType(f), file_path: f, name: extractNodeName(f), metadata: {} }`
7. Build edges array: For each file's imports that resolve to another file in the nodes list, create edge per D-12: `{ from: sourceNodeId, to: targetNodeId, type: inferEdgeType(sourceType, targetType) }`
   - `inferEdgeType`: route->view = "renders", view->component = "renders", component->endpoint = "calls", endpoint->service = "calls", endpoint->model = "uses_table", service->model = "uses_table", default = "imports"
8. Construct tree object: `{ area: slug, generated_at: new Date().toISOString(), nodes, edges }`
9. Write tree.json via `writeTreeJson(cwd, slug, tree)`
10. Update registry entry: set `last_scanned` to now, `components_count` to nodes.length. Write registry.
11. `output({ success: true, area: slug, nodes: nodes.length, edges: edges.length, tree_path: path.join('.planning/ops', slug, 'tree.json') }, raw)`

# /ops:init

Scan codebase and build the OPS system registry.

## Usage

`/ops:init`

## What it does

1. Scans codebase for route definitions (Vue Router, Laravel, Express) and directory conventions (views/, pages/, features/, modules/)
2. Auto-detects functional areas (features/screens) with confidence scoring
3. Deduplicates areas detected by both route and directory methods
4. Creates `.planning/ops/registry.json` with all detected areas
5. Reports detection results with area count and sources

## Implementation

```bash
node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" ops init
```

## Output

JSON with `{ success, areas_detected, areas: [...] }`. Each area contains slug, name, source, detected_by, confidence.

## Notes

- Re-running overwrites the existing registry (fresh scan)
- Does NOT create per-area directories (lazy creation on first `/ops:map`)
- Use `/ops:add` to register areas not covered by auto-detection
```

**commands/gsd/ops-map.md:**

# /ops:map

Build dependency tree for a specific area.

## Usage

`/ops:map <area>`

## What it does

1. Looks up area in registry by slug
2. Finds all files belonging to the area (route files, views, components, endpoints, services, models)
3. Scans imports/requires to discover dependency connections
4. Builds adjacency list graph with typed nodes and edges
5. Writes tree.json to `.planning/ops/{area}/tree.json`
6. Updates registry with components_count and last_scanned timestamp

## Implementation

```bash
node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" ops map <area-slug>
```

## Output

JSON with `{ success, area, nodes, edges, tree_path }`.

## Node Types

route, view, component, endpoint, service, model, table

## Edge Types

imports, calls, renders, serves, uses_table

## Notes

- Area must exist in registry (run `/ops:init` or `/ops:add` first)
- Creates per-area directory if it does not exist
- Re-running regenerates tree.json (fresh scan of area dependencies)
```
