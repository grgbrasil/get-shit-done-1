Create `get-shit-done/workflows/brainstorm-phase.md` per D-11. This is the main workflow file -- it IS the prompt that drives the brainstorm session. Follow the XML `<step>` structure of discuss-phase.md and discuss-phase-assumptions.md.

**Workflow header:** Brief description explaining this is a GSD-native interactive exploration workflow (per D-05). NOT a subagent -- direct conversation between Claude and the user.

**Steps (10 total, in `<process>` tags):**

**Step 1 - initialize:**
```bash
INIT=$(node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" init phase-op "${PHASE}")
```
Parse JSON. Extract `phase_dir`, `phase_number`, `phase_name`, `has_brainstorm`. If init fails, error and exit.

**Step 2 - check_existing:**
If `has_brainstorm` is true: Ask user via AskUserQuestion with options "Update existing" / "View it" / "Skip". If "Update": continue. If "View": display BRAINSTORM.md then re-ask. If "Skip": exit. Same pattern as discuss-phase's `check_existing_context` step.

**Step 3 - load_prior_context:**
Read PROJECT.md, ROADMAP.md, STATE.md. Read prior CONTEXT.md files (phase < current). Build `<prior_decisions>` context. Same pattern as discuss-phase `load_prior_context`.

**Step 4 - scout_codebase:**
Lightweight grep/glob for phase-relevant files. Read codebase maps if they exist (`.planning/codebase/*.md`). Identify reusable assets. Same pattern as discuss-phase-assumptions `scout_codebase` step.

**Step 5 - explore_domain:**
Interactive Q&A loop per D-06 (one question at a time, NEVER multiple questions in same turn). Read the phase goal from ROADMAP.md. Identify 3-5 key design areas needing exploration.

For each area, ask ONE question at a time using AskUserQuestion:
- Prefer multiple-choice when possible (options parameter)
- Open-ended when the domain requires it
- Adapt follow-up questions based on responses
- Stop exploring an area when intent is clear

Question flow heuristic: Start with highest-impact design decision. 2-4 questions per area. Total ~8-15 questions across all areas. When `workflow.text_mode: true`, present numbered lists instead of AskUserQuestion.

**Step 6 - propose_approaches:**
Per D-07: For each major design decision identified in explore_domain, present 2-3 approaches with trade-offs and a recommendation.

Format each approach block as:
```
### [Decision Area]

**Option A: [Name]** (Recommended)
[Description, 2-3 sentences]
- Pros: [key advantages]
- Cons: [key disadvantages]

**Option B: [Name]**
...
```

Use AskUserQuestion for each decision: header = area name, question = "Which approach?", options = [Option names including "(recommended)"].

**Step 7 - capture_decisions:**
Iterate on design decisions until user is satisfied. For each approach selected in Step 6, record as BD-NN with confidence level (Confident if user was definitive, Likely if user went with recommendation, Unclear if user was hesitant).

Present summary of all captured decisions. Ask user via AskUserQuestion: "These are the decisions so far. Anything to add or change?" with options: "Looks good" / "Change something" / "Add more".

**Step 8 - write_brainstorm:**
Write `{NN}-BRAINSTORM.md` to `${phase_dir}/` using the template from `get-shit-done/templates/brainstorm.md`. Fill in all 4 sections: `<domain_boundary>` from ROADMAP.md, `<design_decisions>` from Step 7, `<tradeoffs_explored>` from Step 6, `<pre_context>` with assumptions (Likely confidence items from decisions) and open questions (any Unclear items or unexplored areas).

**Step 9 - git_commit:**
```bash
node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" commit "docs(${PHASE}): brainstorm phase exploration" --files ${phase_dir}/*-BRAINSTORM.md
```

**Step 10 - confirm_creation:**
Per D-09: Display summary of what was explored and decided. Then output EXACTLY:
```
Next steps:
1. /clear (fresh context window)
2. /gsd:discuss-phase [phase_number]
```
NEVER suggest plan-phase or any other command. The brainstorm feeds discuss-phase, period.

**Anti-patterns to enforce in the workflow text:**
- "NEVER ask multiple questions in the same turn" (per D-06)
- "NEVER use Task tool or spawn subagents" (per D-05)
- "NEVER suggest plan-phase as next step" (per D-09)
- "NEVER treat brainstorm decisions as locked -- they are Likely confidence" (per D-04)
