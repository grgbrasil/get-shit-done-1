    Create `hooks/gsd-phase-lock.js` as a PreToolUse hook. Follow the stdin parsing pattern from `gsd-workflow-guard.js` exactly (stdin timeout 3000ms, setEncoding utf8, data/end events, JSON.parse in try/catch with exit(0) on any error).

    Header comment block:
    ```
    #!/usr/bin/env node
    // gsd-hook-version: {{GSD_VERSION}}
    // GSD Phase Lock -- PreToolUse hook
    // Detects concurrent sessions writing to the same phase and BLOCKS the second session.
    // HARD guard -- blocks, not advises. Per D-04 from 05-CONTEXT.md.
    ```

    Logic flow (per D-01 through D-13):

    1. Parse stdin JSON. Extract `tool_name` and `tool_input`.
    2. If `tool_name` is NOT `Write` and NOT `Edit`: `process.exit(0)` (allow reads, bash, etc.)
    3. Extract `filePath` from `data.tool_input.file_path || data.tool_input.path || ''`
    4. Match filePath against regex `/\.planning[\/\\]phases[\/\\](\d+(?:\.\d+)*-[^\/\\]+)/`. If no match: `process.exit(0)` (not a planning phase file, per D-02)
    5. Build `phaseDir = path.join(cwd, '.planning', 'phases', phaseDirName)` where `cwd = data.cwd || process.cwd()`
    6. Extract `phaseNum` from `phaseDirName.match(/^(\d+(?:\.\d+)*)/)?.[1]`
    7. Set `sessionPid = process.ppid` (per D-03)
    8. Build `lockPath = path.join(phaseDir, '.lock')`

    Auto-acquire logic (D-08):
    9. If lockPath does NOT exist AND phaseDir exists: try `fs.writeFileSync(lockPath, JSON.stringify({ pid: sessionPid, acquired: new Date().toISOString() }), { flag: 'wx' })`. On EEXIST: fall through to check. On other errors: `process.exit(0)` (don't block on internal errors).
    10. If lockPath still does not exist: `process.exit(0)` (no lock, phase dir may not exist yet -- allow scaffolding)

    Lock check:
    11. Read and parse lock JSON. On corrupt JSON: `process.exit(0)` (allow -- don't block on corruption)
    12. If `lockData.pid === sessionPid`: `process.exit(0)` (same session, per D-03)
    13. Check if owner is alive: `process.kill(lockData.pid, 0)`. On ESRCH (dead): attempt stale recovery -- unlink old lock, write new with sessionPid via `{ flag: 'wx' }`. If wx succeeds: `process.exit(0)`. If EEXIST: another session grabbed it -- re-read and continue to block check.
    14. Re-read lock after stale recovery attempt. If `lockData.pid === sessionPid`: `process.exit(0)` (acquired via stale recovery)

    Block (D-04, D-13):
    15. Output blocking JSON:
    ```javascript
    {
      hookSpecificOutput: {
        hookEventName: "PreToolUse",
        decision: "block",
        reason: `PHASE LOCK: Phase ${phaseNum} is locked by session PID ${lockData.pid} since ${lockData.acquired}. Another Claude Code session is actively working on this phase. If the other session has crashed, use /gsd:unlock-phase ${phaseNum} to force-release the lock.`
      }
    }
    ```

    Global safety: wrap entire end handler in try/catch with `process.exit(0)` in catch (never block on internal errors).

    ```javascript
    case 'lock': {
      const lock = require('./lib/lock.cjs');
      const subcommand = args[1];
      if (subcommand === 'acquire') {
        lock.cmdLockAcquire(cwd, args[2], args[3], raw);
      } else if (subcommand === 'release') {
        lock.cmdLockRelease(cwd, args[2], raw);
      } else if (subcommand === 'check') {
        lock.cmdLockCheck(cwd, args[2], args[3], raw);
      } else if (subcommand === 'force-unlock') {
        lock.cmdLockForceUnlock(cwd, args[2], raw);
      } else {
        error('Unknown lock subcommand: ' + subcommand + '. Available: acquire, release, check, force-unlock');
      }
      break;
    }
    ```

    ```markdown
    # /gsd:unlock-phase

    Force-release a phase lock when the owning session has crashed or the PID has been reused.

    ## Usage

    `/gsd:unlock-phase <phase-number>`

    ## Process

    1. Resolve phase directory from phase number
    2. Check if lock exists and show current lock info (PID, timestamp)
    3. Force-remove the .lock file
    4. Confirm unlock to user

    ## Implementation

    ```bash
    PHASE_DIR=$(node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" find-phase "$PHASE_NUMBER" 2>/dev/null)
    # Check current lock status
    node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" lock check "$PHASE_DIR"
    # Force unlock
    node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" lock force-unlock "$PHASE_DIR"
    ```

    ## Notes

    - This is an escape hatch for edge cases (crashed sessions, PID reuse)
    - Normal operation: locks are auto-acquired by the hook and auto-released when sessions end
    - Use when you see "PHASE LOCK" blocking messages but know the other session is no longer active
    ```
