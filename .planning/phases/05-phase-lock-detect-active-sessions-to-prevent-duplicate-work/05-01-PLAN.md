  <behavior>
    - acquire(phaseDir, pid): creates .lock with JSON `{ pid: Number, acquired: ISO-string }` using `{ flag: 'wx' }`, returns `{ acquired: true }`
    - acquire(phaseDir, pid): when .lock exists and owner PID alive, returns `{ acquired: false, owner_pid: Number, acquired_at: string }`
    - acquire(phaseDir, pid): when .lock exists and owner PID dead (ESRCH), unlinks stale lock, creates new, returns `{ acquired: true, was_stale: true }`
    - acquire(phaseDir, pid): when .lock is corrupt JSON, unlinks it, creates new, returns `{ acquired: true, was_corrupt: true }`
    - acquire(phaseDir, pid): race condition during stale recovery (second wx fails EEXIST), re-reads lock and returns `{ acquired: false, owner_pid, acquired_at }`
    - release(phaseDir): unlinks .lock, returns `{ released: true }`
    - release(phaseDir): when no .lock exists, returns `{ released: true, was_absent: true }`
    - check(phaseDir, currentPid): when no .lock, returns `{ locked: false }`
    - check(phaseDir, currentPid): when .lock exists with live PID, returns `{ locked: true, owner_pid, acquired_at, stale: false, own_session: bool }`
    - check(phaseDir, currentPid): when .lock exists with dead PID, returns `{ locked: true, ..., stale: true }`
    - check(phaseDir, currentPid): when .lock corrupt, returns `{ locked: false, corrupt: true }`
    - forceUnlock(phaseDir): removes .lock regardless, same return as release()
    - isProcessAlive(pid): returns true for own process, false for non-existent PID (ESRCH), true for EPERM
    - LOCK_FILENAME equals '.lock'

    GREEN phase: Create `get-shit-done/bin/lib/lock.cjs` as CommonJS module. Follow the exact pattern from 05-RESEARCH.md code example. Key implementation details:

    1. `isProcessAlive(pid)`: `process.kill(pid, 0)` wrapped in try/catch. Returns `false` on ESRCH, `true` on EPERM, `false` on other errors.

    2. `acquire(phaseDir, pid)`: Build lockPath as `path.join(phaseDir, '.lock')`. JSON data: `{ pid: Number(pid), acquired: new Date().toISOString() }`. Use `fs.writeFileSync(lockPath, data, { flag: 'wx' })`. On EEXIST: read existing, check `isProcessAlive(existing.pid)`. If alive: return `{ acquired: false, owner_pid, acquired_at }`. If dead: unlink, retry wx (wrap retry in try/catch for EEXIST race -- if lost, re-read and return false). On corrupt JSON: unlink, retry wx, return `{ acquired: true, was_corrupt: true }`.

    3. `release(phaseDir)`: `fs.unlinkSync(lockPath)`. ENOENT catch returns `{ released: true, was_absent: true }`.

    4. `check(phaseDir, currentPid)`: `fs.existsSync` first. If no lock: `{ locked: false }`. Read and parse JSON (corrupt: `{ locked: false, corrupt: true }`). Check `isProcessAlive(data.pid)` for stale. Compare `Number(currentPid) === data.pid` for own_session.

    5. `forceUnlock(phaseDir)`: delegates to `release(phaseDir)`.

    6. CLI wrappers: `cmdLockAcquire(cwd, phaseDir, pid, raw)`, `cmdLockRelease(cwd, phaseDir, raw)`, `cmdLockCheck(cwd, phaseDir, currentPid, raw)`, `cmdLockForceUnlock(cwd, phaseDir, raw)`. Each resolves relative paths with `path.isAbsolute(phaseDir) ? phaseDir : path.join(cwd, phaseDir)`, validates required args with `error()`, calls core function, outputs via `output()`.

    7. Exports: `{ acquire, release, check, forceUnlock, isProcessAlive, LOCK_FILENAME, cmdLockAcquire, cmdLockRelease, cmdLockCheck, cmdLockForceUnlock }`
