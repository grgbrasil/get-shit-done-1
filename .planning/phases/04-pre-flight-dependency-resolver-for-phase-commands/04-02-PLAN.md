<objective>
Wire the preflight resolver into GSD phase workflows as an early validation gate.

Purpose: Each phase workflow (plan-phase, execute-phase, ui-phase) gains a preflight check immediately after init. When prerequisites are missing, the workflow displays the exact command to run and exits cleanly. Existing inline checks are preserved -- preflight augments, does not replace them. This creates a consistent first-pass gate across all workflows.

Output: Three modified workflow files with preflight integration steps.
</objective>

<interfaces>
<!-- Preflight command interface from Plan 01 -->

CLI invocation:
```bash
node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" preflight <command> <phase>
# command: plan-phase | execute-phase | ui-phase
# phase: phase number (e.g., "4", "01")
```

Output JSON shape:
```json
{
  "ready": true|false,
  "blockers": [
    {
      "type": "missing_context|missing_ui_spec|incomplete_dependency|no_plans",
      "message": "Human-readable description",
      "action": "run_command",
      "command": "/gsd:discuss-phase 4",
      "severity": "blocking|warning",
      "skippable": true|false
    }
  ],
  "next_action": "run_command"|null,
  "next_command": "/gsd:discuss-phase 4"|null,
  "phase_number": "4",
  "command_checked": "plan-phase"
}
```

Workflow integration pattern (from RESEARCH.md Pattern 4):
```bash
PREFLIGHT=$(node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" preflight plan-phase "${PHASE}" 2>/dev/null)
READY=$(echo "$PREFLIGHT" | node -e "process.stdout.write(JSON.parse(require('fs').readFileSync(0,'utf-8')).ready.toString())")

if [ "$READY" = "false" ]; then
  NEXT_CMD=$(echo "$PREFLIGHT" | node -e "process.stdout.write(JSON.parse(require('fs').readFileSync(0,'utf-8')).next_command || '')")
  echo "Prerequisites not met. Run first:"
  echo ""
  echo "  ${NEXT_CMD}"
  echo ""
  echo "Then re-run /gsd:plan-phase ${PHASE}"
  # EXIT WORKFLOW
fi
```
